More Jquery

$() grabs an object. It can be specified by $("#id")
$("#id").html() is a getter
$("#id").html("dog") is a setter
$("div") selects and grabs all of the divs, and
$("div").each(callback) is a callback function, which fires an event for each div
A callback is a function that is bound to an event handler. It is executed when the event is triggered

AJAX

AJAX stands for Asynchronous JavaScript and XML
AJAX requests are made using the XHR (XMLHttpRequest) browser object
The hard part of XHR are the status codes.
JQuery made XHR easier with AJAX.
There is also the fetch() API, which came out in 2017
With AJAX, you can make requests to a server without reloading the page
AJAX takes a JSON object and sends it to the server
The server processes the JSON object and sends a response back to the client
The client processes the response and updates the page

Sidenote: jquery siblings() = parent().children()

More AJAX

Loading a webpage begins with a synchronous transaction
The browser sends a request to the server, and the server sends a response back
If the user fires an event, the browser sends another request to the server asynchronously
This means the user can interact with the page while the server is processing the request
The server sends a response back to the client as JSON, and the client updates the page

Arrow functions

Arrow functions are a shorthand way to write functions
They are anonymous functions
() => {} is the syntax
It is the same as function() {}
Arrow functions do not have their own this
They inherit the this from the parent scope
You don't always have to use return with arrow functions
If you have a single line of code, you can omit the curly braces
If you have a single parameter, you can omit the parentheses
For example: x => x * x
That is the same as function(x) { return x * x; }

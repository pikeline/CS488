A callback function is a function that is triggered when an event is triggered.
2 ways to assign an event handler to an event:
  1. HTML attribute
    ex: <button onclick="alert('Hello!') onmouseover="f()">Click me</button>
  2. JS property
    ex:
    document.getElementById('myButton').onclick = function() {
    alert('Hello!');
    }
     or, you can do it using addEventListener: (new way)
    document.getElementById('myButton').addEventListener('click', function() {
    alert('Hello!');
    });
Events that fire, in order:
  1. DOMContentLoaded event fires on the document object.
    It means the DOM is ready, so you can safely access elements. You can't use stuff like getElementById() before this event
  2. window.onload event fires on the window object.
DOM vs BOM
The DOM is the Document Object Model, which is a tree-like structure that represents the elements of an HTML document.
The BOM is the Browser Object Model, which is a tree-like structure that represents the browser window.
Browser objects are things like window, screen, location, history, navigator, etc.
Document objects are everything in the document.
You can:
  1. change the innerHTML of an element
  2. change classnames with the className property
    Note: the class name has to already exist.
  3. change the styles that exist in the style attribute
    Note: you have to write styles in camel case. For example, backgroundColor instead of background-color

JSON and DOM Factory

Elements can be created using the DOM factory methods.
For example, document.createElement('div') creates a new div element.
Attributes can be added to elements using the setAttribute() method.
The element will begin to exist when it is appended to the DOM with appendChild().

With events, Javascript will automatically pass the event object to the callback function.
For example, document.getElementById('myButton').onclick = function(e) { console.log(e); }
or, in an attribute, <a onclick=f()"></a>
JSON is a text-based protocol that sends plain-text UTF-8 data.
It is serialization of objects into strings.
Serialization means conversion of objects into strings.
This allows for easy passing of objects between systems via http.

XML vs JSON

XML was Microsoft's version of JSON. XML makes you invent a markup language. It isn't that efficient to transmit via text, since it has a lot of tags. JSON is more efficient, since it is just key-value pairs.
In JSON, {} is an object, and [] is an array, and , separates properties.
In JavaScript, you can use backticks ` to create multiple line strings.
Inside JSON, you need to use double quotes.

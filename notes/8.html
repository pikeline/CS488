CRUDL - Create, Read, Update, Delete, List
Create - INSERT INTO table (column1, column2, ...) VALUES (value1, value2, ...)
Read - SELECT * FROM table WHERE column = value
Update - UPDATE table SET column1 = value1, column2 = value2 WHERE column = value
Delete - DELETE FROM table WHERE column = value
List - SELECT * FROM table
The form data should have a hidden input field called task, which passes a value called save, delete, edit, or blank(default) and a field called name with a value of id, which is the primary key if it exists (empty if new, existing if upgrade). In the serverside, there is a switch statement that checks the value of task and performs the appropriate action.
save cleans the data, then either inserts or updates the data in the database.
Finally, feedback is given.

Why post?

If you refresh a get string, it will resubmit the data. If you refresh a post string, it will not resubmit the data. This is why post is used for forms.

Why use a hidden input field?

It is a way to pass data that the user cannot see. It is used to pass the task and the id of the data.
It is also useful to pass data that is not user input, such as the date and time of the submission.
In addition, spreading code across pages is useful for compartmentalizing code.

MVC

Model View Controller is a way to organize code. The model is the database, the view is the user interface, and the controller is the code that connects the two.
For example, the controller can be a php file that connects to the database and the view can be an html file that displays the data.
Form data is passed to a controller.php file, which then updates the model and then returns to the view.

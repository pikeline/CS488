Javascript - Vanilla edition
Vanilla javascript is javascript without any libraries or frameworks.
W3C controls HTML, CSS. ECMAScript controls Javascript.
The current version of Javascript is 6.
Javascript is loosely typed, which means you don't have to declare the type of a variable when you declare it.
ex:
var x;
x = 10;
x = "hello";

it doesn't care

while, Java is strongly typed, which means you have to declare the type of a variable when you declare it.
ex:

javascript is interprited on the fly, which means it is executed line by line.
Java is compiled, which means it is converted to machine code before it is executed.

Historically, JS talked with the browser's API for client side scripting. It is used for more now.
Script = code to control an app
Code = making the app
JS made the jump to server side with Node JS. It was the original server side library.
http(s) is called a stateless protocol. It doesn't remember anything about the client.
The client sends a request, the server sends a response, and then forgets about the client.
If the client needs something, it asks the server, the server sends it, then the server closes the requests.
There is no lasting connection
Client pull means the client dictates the request, and the server responds.
Nowadays, there is websockets, which is a two way connection.
It is called websocket secure because it is encrypted.
It is a stateful, bi-directional protocol aka server push and client pull.
So, when a webpage loads, it can keep the connection open and the server can push data to the client.

Javascript was invented by Netscape browser. They also invented cookies, which is a little chunk of data set by the browser.

Every browser has a javascript engine. Chrome has V8, Firefox has SpiderMonkey, and Edge has Chakra.

Errors are dumped to the console.

Sequence of events when a Page loads

  1. Load time execution.
    This sets up event handlers and other initial tasks like creating connections
  2. document.DOMContentLoaded event
    DOM becomes fully initialized and loaded. You can not access html objects before this.
  3. window.onload
    This is when the page is fully loaded. You can access all html objects now.
  4. Event loop
    This is a loop that checks every millisecond for events. It is always running.
HTML objects are every element in the page. They are stored in a tree structure.
DHTML is dynamic HTML. It refers to manipulating the DOM with javascript.
=> you can do dynamic things in pages.
It was a marketing term.

When a function is tied to an event, it is called a callback function.
These can be named or anonymous.
Anonymous functions are functions without a name.

What can you change about an HTML object

You can change it's content, style, and class.
ex:
obj.innerHTML = "new content";
obj.style.backgroundColor = "red";
obj.className = "new_class";
Note: "new_class" must pre_exist in the style sheet.

You can use the ability to modify styles to adfd animations to a page.
Ex: add 1 pixel to the width of a div every 10 milliseconds.

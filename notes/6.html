ReactJS is a CSR (Client Side Rendering) library that is used to build user interfaces.
It uses JSX, which is JSXML, a syntax extension of JavaScript.
JSX is a syntax that allows HTML to be written in JavaScript and placed in the same file.
It must be legit XML, like self-closing tags and lowercase tag names.
Also, the tags must be closed in the proper order.

For example, the following code snippet is a React component that renders a simple greeting:
        import React from 'react';
        import ReactDOM from 'react-dom';

        function Greeting() {
            return <>
                Hello, World! 

                Hi!
            <>;
        }

        ReactDOM.render(, document.getElementById('root'));
React creates a Virtual DOM parse tree that is a lightweight copy of the actual DOM.
When the state of the Virtual DOM changes, React compares the Virtual DOM with the actual DOM and only updates the parts that have changed.
The Javascript html is used to build a Virtual DOM tree.
React also has a global state.
It gives the current state of a component
User events are a change in the global state.
The react virtual dom has something called react hooks, which are functions that let you use state and other React features without writing a class.

Events are caused by user actions like clicking a button or typing in a form or web sockets.
Web sockets allow servers to push data to clients in real time.
When there is a change, react builds a new virtual dom and compares it to the old one.
It then updates the actual dom with the changes using the React Fiber algorithm.
This is different from shadow DOM, which is a way to create a separate DOM tree that is hidden from the main DOM.
For example, the video element in HTML uses shadow DOM.
The controls are in the shadow DOM and are not accessible to the main DOM.
